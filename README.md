# Веб-система для управления мероприятиями 🎓

Данный репозиторий содержит исходный код проекта, разрабатываемого в рамках дипломной работы (выпускной квалификационной работы).
Проект представляет собой полнофункциональное веб-приложение с раздельной клиент-серверной архитектурой, предоставляющее RESTful API для организации мероприятий и проведения опросов.

## 🚀 Стек технологий

## ✨ Возможности

* 🔐 Регистрация/вход (JWT), автогенерация **@username**, личный кабинет с редактированием профиля
* 📅 Мероприятия: создание, удаление, статусы
* 👥 **Приглашения** других пользователей в мероприятие по @username/email с принятием/отклонением
* 🔔 **Уведомления**: приветствие при регистрации, приглашения (видно, кто пригласил), системные события, бейдж непрочитанных
* 🛡️ **Админка**: список пользователей и **массовая рассылка** уведомлений всем
* 🤖 ИИ-ассистент поддержки (DeepSeek)
* 🎬 Анимации интерфейса (Framer Motion)

> Первый зарегистрированный пользователь автоматически становится **администратором** (или любой email из переменной `ADMIN_EMAILS`).

## 🚀 Стек технологий

### Серверная часть (Backend):
* **Фреймворк:** FastAPI (Python 3.13)
* **База данных:** PostgreSQL + SQLAlchemy (асинхронный движок, asyncpg)
* **Миграции:** Alembic (накатываются автоматически при старте контейнера)
* **Кэширование:** Redis (кэш списка мероприятий с инвалидацией)
* **Авторизация:** JWT (PyJWT) + хэширование паролей PBKDF2
* **ИИ-поддержка:** DeepSeek (OpenAI-совместимый API)
* **Контейнеризация:** Docker & Docker Compose

### Клиентская часть (Frontend):
* **Библиотека:** React 19 + Vite
* **Стилизация:** Tailwind CSS
* **Раздача в проде:** Nginx (проксирует `/api` на бэкенд)

## 📂 Структура проекта

* `backend/` — Асинхронное REST API
  * `app/api/v1/` — Роутеры (auth, events, chat)
  * `app/models/` — ORM-модели (User, Event)
  * `app/schemas/` — Схемы валидации (Pydantic)
  * `app/core/` — Конфиг, БД, безопасность
* `frontend/` — Клиентское одностраничное приложение (SPA)
* `docker-compose.yml` — Оркестрация всех сервисов

## 🐳 Запуск через Docker (полный цикл)

Поднимает разом **PostgreSQL + Backend + Frontend** одной командой.

1. Убедитесь, что есть файл `backend/.env` с ключом DeepSeek:
   ```env
   DEEPSEEK_API_KEY=sk-...
   ```
2. Из каталога `eventshandler/`:
   ```bash
   docker compose up --build -d
   ```
3. Откройте приложение: **http://localhost:5173**

| Сервис   | Адрес                          | Назначение                    |
|----------|--------------------------------|-------------------------------|
| Frontend | http://localhost:5173          | SPA (Nginx)                   |
| Backend  | http://localhost:8000/docs     | REST API + Swagger            |
| Postgres | localhost:5432 (events/events) | База данных                   |
| Redis    | внутренняя сеть (redis:6379)   | Кэш                           |

> Миграции Alembic (`alembic upgrade head`) запускаются автоматически перед стартом бэкенда — схема БД создаётся из миграций, а не «на лету».

Полезные команды:
```bash
docker compose logs -f backend   # логи бэкенда
docker compose ps                # статус контейнеров
docker compose down              # остановить
docker compose down -v           # остановить и стереть данные БД
```

> Таблицы создаются автоматически при старте бэкенда. Данные Postgres хранятся в Docker-томе `pgdata` и переживают перезапуск.

## 🌐 Деплой на сервер

1. Установите на сервер Docker и Docker Compose.
2. Склонируйте репозиторий и перейдите в `eventshandler/`.
3. Создайте секреты:
   ```bash
   cp .env.example .env                 # переменные compose (SECRET_KEY, пароль БД, порты, домен)
   cp backend/.env.example backend/.env # ключ DeepSeek
   ```
   В `.env` **обязательно** задайте надёжный `SECRET_KEY`, `POSTGRES_PASSWORD`, домен в `CORS_ORIGINS`
   и `FRONTEND_PORT=80`. В `backend/.env` укажите `DEEPSEEK_API_KEY`.
4. Запустите:
   ```bash
   docker compose up -d --build
   ```
   Все сервисы имеют `restart: unless-stopped` и поднимутся автоматически после перезагрузки сервера.
5. Откройте `http://<домен-или-IP>` — первый зарегистрированный пользователь станет администратором.

**Рекомендации для прод-сервера:**
- Поставьте перед фронтендом обратный прокси (Nginx/Caddy/Traefik) с HTTPS (Let's Encrypt).
  Фронтенд уже сам проксирует `/api` на бэкенд, поэтому наружу достаточно одного порта 80/443.
- Не публикуйте порт Postgres наружу — закройте `DB_PORT` файрволом или уберите проброс.
- Миграции применяются автоматически; данные — в томе `pgdata` (бэкап: `docker compose exec db pg_dump …`).

## 💻 Локальный запуск без Docker (для разработки)

**Бэкенд:**
```bash
cd backend
python -m venv .venv && .venv\Scripts\activate   # Windows
pip install -r requirements.txt
alembic upgrade head        # создать схему БД
uvicorn app.main:app --reload --port 8000
```
По умолчанию используется локальная SQLite-база (`events.db`) — Postgres поднимать не обязательно.
Redis в dev-режиме не требуется: если `REDIS_URL` не задан, кэш просто отключается.

**Фронтенд:**
```bash
cd frontend
npm install
npm run dev
```
В dev-режиме фронт обращается к API на `http://localhost:8000` (нужен запущенный бэкенд).
